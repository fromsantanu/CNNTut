# 📘 Chapter 10: Training the Model

---

## 🧠 10.1 What Does “Training” Mean?

Building the model is only half the job.
Now we must **teach it to recognize digits**.

👉 Training means:

> Adjusting model weights so predictions become correct

---

### 🧠 Real-life analogy

Like a student:

* Makes mistakes → gets feedback → improves

👉 Model does the same:

* Predict → compare → correct

---

## 🔄 10.2 Training Loop Overview

```text
Input Image
   ↓
Model Prediction
   ↓
Compare with True Label (Loss)
   ↓
Adjust Weights (Backpropagation)
   ↓
Repeat many times (Epochs)
```

---

## ⚙️ 10.3 Step 1: Define Loss Function

The loss tells:

> “How wrong is the model?”

---

### ✅ For classification:

```python
criterion = nn.CrossEntropyLoss()
```

👉 Works well for:

* Digit classification (0–9)

---

## 🔧 10.4 Step 2: Define Optimizer

Optimizer updates weights.

```python
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
```

---

👉 Think of optimizer as:

> “How fast and in what direction to learn”

---

## 🔁 10.5 Step 3: Training Loop (Core Part)

```python
epochs = 5

for epoch in range(epochs):
    for images, labels in train_loader:
        
        # Forward pass
        outputs = model(images)
        
        # Calculate loss
        loss = criterion(outputs, labels)
        
        # Clear previous gradients
        optimizer.zero_grad()
        
        # Backpropagation
        loss.backward()
        
        # Update weights
        optimizer.step()
    
    print(f"Epoch [{epoch+1}/{epochs}], Loss: {loss.item():.4f}")
```

---

## 🧠 10.6 Understanding Each Step

---

### 🔹 Forward Pass

```python
outputs = model(images)
```

👉 Model makes prediction

---

### 🔹 Loss Calculation

```python
loss = criterion(outputs, labels)
```

👉 Measures error

---

### 🔹 Zero Gradients

```python
optimizer.zero_grad()
```

👉 Clears old learning

---

### 🔹 Backpropagation

```python
loss.backward()
```

👉 Finds how to improve

---

### 🔹 Update Weights

```python
optimizer.step()
```

👉 Applies learning

---

## 🔄 10.7 What is an Epoch?

* One complete pass over dataset

---

👉 Example:

* 60,000 images
* 1 epoch = model sees all once

---

👉 More epochs:

* More learning
* But risk of overfitting

---

## 📉 10.8 Monitoring Loss

During training:

* Loss should decrease

---

👉 Example:

```
Epoch 1 → Loss: 0.8  
Epoch 2 → Loss: 0.4  
Epoch 3 → Loss: 0.2  
```

---

👉 If loss not decreasing:

* Learning rate issue
* Model problem

---

## 📊 10.9 Tracking Accuracy (Optional but Important)

```python
correct = 0
total = 0

with torch.no_grad():
    for images, labels in train_loader:
        outputs = model(images)
        _, predicted = torch.max(outputs, 1)
        
        total += labels.size(0)
        correct += (predicted == labels).sum().item()

print("Accuracy:", 100 * correct / total)
```

## ⚠️ 10.11 Common Mistakes

### ❌ Not using zero_grad()

* Gradients accumulate

---

### ❌ Too high learning rate

* Model becomes unstable

---

### ❌ Too few epochs

* Model doesn’t learn

---

## 🧪 10.12 Full Training Code (Combined)

```python
import torch
import torch.nn as nn
import torch.optim as optim

criterion = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=0.001)

epochs = 5

for epoch in range(epochs):
    for images, labels in train_loader:
        
        outputs = model(images)
        loss = criterion(outputs, labels)
        
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
    
    print(f"Epoch {epoch+1}, Loss: {loss.item():.4f}")
```
