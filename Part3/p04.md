# 📘 Chapter 11: Model Evaluation

---

## 🧠 11.1 Why Evaluation is Important

After training, a key question remains:

> “How good is our model on **new, unseen data**?”

👉 Training performance alone is not enough.
👉 We must test the model on **test data**.

---

### 🧠 Real-life analogy

A student may:

* Practice well at home (training)
* But real test happens in exam (testing)

👉 Evaluation = **final exam of the model**

---

## 🔄 11.2 Train vs Test Data

* **Training data** → used to learn
* **Test data** → used to evaluate

👉 Important:

> Test data must be unseen during training

---

## 📊 11.3 Accuracy (Most Basic Metric)

### ✅ What is Accuracy?

> Percentage of correct predictions

---

### 📌 Formula

\text{Accuracy} = \frac{\text{Correct Predictions}}{\text{Total Predictions}}

---

### 🧠 Example

* Correct = 90
* Total = 100

👉 Accuracy = 90%

---

## 🧪 11.4 Evaluate Model in PyTorch

```python
correct = 0
total = 0

with torch.no_grad():  # no training
    for images, labels in test_loader:
        outputs = model(images)
        _, predicted = torch.max(outputs, 1)
        
        total += labels.size(0)
        correct += (predicted == labels).sum().item()

accuracy = 100 * correct / total
print(f"Test Accuracy: {accuracy:.2f}%")
```

---

👉 `torch.no_grad()`:

* Makes evaluation faster
* No gradient calculation

---

## 🧠 11.5 What is a Good Accuracy?

For **MNIST dataset**:

* 95% → good
* 98%+ → very good

---

👉 But always remember:

> High accuracy ≠ perfect model

---

## 📉 11.6 When Accuracy Misleads

Accuracy can sometimes **hide problems**.

---

### 🧠 Example

Suppose:

* 90% images are “0”
* Model predicts always “0”

👉 Accuracy = 90%
👉 But model is useless

## 🧪 11.8 Confusion Matrix (PyTorch + sklearn)

```python
from sklearn.metrics import confusion_matrix
import numpy as np

all_preds = []
all_labels = []

with torch.no_grad():
    for images, labels in test_loader:
        outputs = model(images)
        _, predicted = torch.max(outputs, 1)
        
        all_preds.extend(predicted.numpy())
        all_labels.extend(labels.numpy())

cm = confusion_matrix(all_labels, all_preds)
print(cm)
```

---

## 🔍 11.9 Precision and Recall (Simple Idea)

We won’t go deep now, just intuition:

* **Precision** → How many predicted are correct
* **Recall** → How many actual were found

---

👉 Important for:

* Medical data
* Imbalanced datasets

---

## 🧠 11.10 Checking Individual Predictions

```python
import matplotlib.pyplot as plt

image, label = test_loader.dataset[0]

output = model(image.unsqueeze(0))
_, predicted = torch.max(output, 1)

plt.imshow(image.squeeze(), cmap='gray')
plt.title(f"True: {label}, Predicted: {predicted.item()}")
plt.show()
```

---

👉 Helps students:

* See what model is doing
* Understand mistakes

---

## ⚠️ 11.11 Common Evaluation Mistakes

### ❌ Evaluating on training data

* Gives false confidence

---

### ❌ Not using `no_grad()`

* Slower performance

---

### ❌ Ignoring wrong predictions

* Misses learning opportunity

---

## 🎯 11.12 Key Takeaways

* Evaluation checks real performance
* Accuracy is basic but useful
* Confusion matrix gives deeper insight
* Always test on unseen data

---

👉 Most important idea:

> A good model must perform well on data it has never seen
