# 📘 Chapter 14: Hyperparameter Tuning

---

## 🧠 14.1 What is Hyperparameter Tuning?

So far, we built and trained a CNN.
But one important question remains:

> “How do we choose the *best settings* for our model?”

These settings are called **hyperparameters**.

---

### 👉 What is a Hyperparameter?

A hyperparameter is something:

* You **set before training**
* The model does **not learn automatically**

---

### 🧠 Real-life analogy

Cooking a dish:

* Ingredients = data
* Recipe = model
* **Spices amount = hyperparameters**

👉 Too much or too little → bad result

---

## ⚙️ 14.2 Common Hyperparameters in CNN

---

### 🔹 1. Learning Rate

* Controls how fast model learns

```python
lr = 0.001
```

👉 Too high → unstable
👉 Too low → very slow

---

### 🔹 2. Number of Epochs

* How many times model sees data

👉 More epochs → more learning
👉 Too many → overfitting

---

### 🔹 3. Batch Size

* Number of samples processed at once

```python
batch_size = 32
```

👉 Small batch → stable but slow
👉 Large batch → fast but less accurate sometimes

---

### 🔹 4. Number of Filters

* In convolution layers

```python
nn.Conv2d(1, 8, 3)
```

👉 More filters → more feature learning
👉 But more computation

---

### 🔹 5. Kernel Size

* Size of filter

👉 Common:

* 3×3
* 5×5

---

### 🔹 6. Dropout Rate

```python
Dropout(p=0.5)
```

👉 Controls regularization

---

## 📊 14.3 Why Tuning is Important

Same model, different settings:

| Setting     | Accuracy |
| ----------- | -------- |
| Poor choice | 85%      |
| Good choice | 98%      |

---

👉 Huge difference just by tuning!

---

## 🔍 14.4 How to Tune Hyperparameters

---

### ✅ 1. Manual Tuning (Beginner Friendly)

Try different values:

```text
Learning Rate: 0.01 → 0.001 → 0.0001
Batch Size: 16 → 32 → 64
```

👉 Observe performance

---

### ✅ 2. Grid Search (Systematic)

Try all combinations:

```text
LR = [0.01, 0.001]
Batch = [32, 64]
```

👉 Test all possibilities

---

### ✅ 3. Random Search

Pick random combinations

👉 Faster than grid search

## 🧠 14.6 Practical Strategy (Very Important)

Follow this order:

---

### Step 1: Fix model architecture

---

### Step 2: Tune learning rate first

---

### Step 3: Adjust batch size

---

### Step 4: Tune epochs

---

### Step 5: Add dropout if needed

---

👉 Don’t change everything at once!

---

## 🧪 14.7 Example (PyTorch)

```python
learning_rates = [0.01, 0.001, 0.0001]

for lr in learning_rates:
    optimizer = torch.optim.Adam(model.parameters(), lr=lr)
    
    print(f"Training with lr={lr}")
    
    # Train model (short run)
```

## ⚠️ 14.8 Common Mistakes

### ❌ Changing too many things together

👉 Cannot understand what worked

---

### ❌ Not tracking results

👉 Always record experiments

---

### ❌ Ignoring validation data

👉 Leads to overfitting

---

## 🧠 14.9 Key Insight

There is **no perfect setting** for all problems.

👉 Best hyperparameters depend on:

* Dataset
* Model
* Task

---

## 🎯 14.10 Key Takeaways

* Hyperparameters control training behavior
* Small changes can give big improvements
* Tuning is a trial-and-error process

---

👉 Most important idea:

> Good model + good hyperparameters = best performance
