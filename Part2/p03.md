# 📘 Chapter 6: Pooling Layer

---

## 🧠 6.1 Why Do We Need Pooling?

After convolution + ReLU, we get **feature maps**.
But these maps are still **large in size**.

👉 Problems if we keep them as-is:

* Too much computation
* Slow training
* Risk of overfitting

---

👉 Solution:

> **Pooling layer reduces size while keeping important information**

---

## 🔍 6.2 Simple Intuition (Real-Life Example)

Imagine you read a long paragraph:

* You don’t memorize every word
* You remember only **key points**

👉 Pooling does the same:

* Keeps important values
* Removes unnecessary details

## 🔹 6.4 Types of Pooling

---

### ✅ 1. Max Pooling (Most Important)

👉 Takes the **maximum value**

---

### 🧠 Example

Input (2×2 region):

```id="xq9n0g"
[1   5
 2   3]
```

Output:

```id="zjz1v7"
5
```

👉 Keeps strongest feature

---

### ✅ 2. Average Pooling

👉 Takes the **average value**

---

Example:

```id="yvl5ah"
(1 + 5 + 2 + 3) / 4 = 2.75
```

---

👉 In practice:

* **Max pooling is used most of the time**

---

## 🔄 6.5 Step-by-Step Pooling Example

### Input Feature Map (4×4)

```id="k0h1s6"
1   3   2   4
5   6   1   2
7   2   8   3
1   0   2   4
```

---

### Apply 2×2 Max Pooling

Divide into blocks:

```id="d8um7j"
[1 3 | 2 4]
[5 6 | 1 2]

[7 2 | 8 3]
[1 0 | 2 4]
```

---

### Take max from each block

```id="m3p4vs"
6   4
7   8
```

---

👉 Output size reduced from **4×4 → 2×2**

---

## ⚙️ 6.6 Important Parameters

---

### 🔹 1. Pool Size

* Size of region
* Common: **2×2**

---

### 🔹 2. Stride

* How much we move

👉 Usually:

```
Stride = Pool size
```

---

## 🧠 6.7 Why Pooling is Powerful

### ✅ 1. Reduces Size

* Less computation
* Faster training

---

### ✅ 2. Keeps Important Features

* Keeps strongest signals

---

### ✅ 3. Provides Translation Invariance

👉 Even if object moves slightly:

* CNN still detects it

---

### 🧠 Example

If a feature shifts:

* Max pooling still captures it

## 🧪 6.9 Hands-on Example (Python)

```python id="2bb7yl"
import numpy as np

# Feature map
feature_map = np.array([
    [1, 3, 2, 4],
    [5, 6, 1, 2],
    [7, 2, 8, 3],
    [1, 0, 2, 4]
])

# Max pooling (2x2)
output = np.zeros((2, 2))

for i in range(0, 4, 2):
    for j in range(0, 4, 2):
        block = feature_map[i:i+2, j:j+2]
        output[i//2, j//2] = np.max(block)

print("Pooled Output:\n", output)
```

---

## 🔬 6.10 PyTorch Example

```python id="k7j9q2"
import torch
import torch.nn as nn

pool = nn.MaxPool2d(2)

x = torch.tensor([[[[1., 3., 2., 4.],
                    [5., 6., 1., 2.],
                    [7., 2., 8., 3.],
                    [1., 0., 2., 4.]]]])

output = pool(x)

print(output)
```

* Max pooling is most commonly used
* Helps CNN become efficient and robust

👉 Most important idea:

> Pooling = **keep important, discard unnecessary**
