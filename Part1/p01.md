# 📘 Chapter 1: Introduction to Images in Machine Learning

---

## 🧠 1.1 What is an Image (in simple terms)?

When we look at an image, we see objects like faces, trees, or animals.
But for a computer, an image is just **numbers arranged in a grid**.

👉 Think of it like an **Excel sheet**:

* Each cell = one number
* Each number = brightness of that pixel

---

## 📷 1.2 Image as a Grid of Pixels

![Image](https://images.openai.com/static-rsc-4/Gznh06EhZq9MXvvBkGSPgZHO2TQu84Tq5gyTDvTE9cxZcTpL0Iww6I6-FyQvGkFAWBmKy0-3NAWEGEc73ym8aEjfjC8oyMwk0fC2REfpgy7Ske7edOpcBlJuNaMu1z3PwO_Ag--ZLF6WXxTzg2hlSLH9Qh413xiPa4k1xy2t0no9oUy5juhYSotIRYSI2MB9?purpose=fullsize)

* Image = collection of **pixels**
* Each pixel has a value:

  * 0 → black
  * 255 → white
  * Between → shades of gray

👉 Example (5×5 image):

```
0   0   50  200 255
10  20  80  180 240
...
```

👉 This is how the computer “sees” an image.

---

## 🎨 1.3 Grayscale vs Color Images

### 🔹 Grayscale Image

* Only one number per pixel
* Shape: **Height × Width**

👉 Example:

* 28 × 28 image → 784 values

---

### 🔹 Color Image (RGB)

![Image](https://images.openai.com/static-rsc-4/gw4eZB2-8mYEiUvOTMOt694y4nqyLw_cLsWQHIQfGnM74A4v6c3aYUs4j7NeOoOjiHHxb3gXwQIfKPCDvuz7-UJwK5Go6S1uuQWckecv8RmKijbBfT8gutYAK9zhn-5_lhxIqZmRWUKvQWCQDzS_fF3WZR5-TPT_j7XUPCwKCTZphCtpoWtZl7vgg1BjKxzi?purpose=fullsize)

* Each pixel has **3 values**:

  * Red
  * Green
  * Blue

👉 Shape becomes:

```
Height × Width × 3
```

👉 Example:

* 64 × 64 × 3 image

---

## 📐 1.4 Image Shape (Very Important Concept)

An image always has a **shape**.

👉 Examples:

* Grayscale:

  ```
  (28, 28)
  ```

* Color:

  ```
  (64, 64, 3)
  ```

👉 Simple meaning:

* Height = number of rows
* Width = number of columns
* Channels = color information

---

## 🔍 1.5 What Machine Learning Sees

Humans see:

> “This is a cat 🐱”

Machine sees:

> Matrix of numbers

👉 Example:

```
[ [123, 45, 67],
  [255, 200, 100],
  ... ]
```

So the model must **learn patterns from numbers**, not from meaning.

---

## ⚠️ 1.6 Why This Matters

This idea is the **foundation of CNNs**.

If you don’t understand this, everything later becomes confusing.

👉 Key idea:

> CNN does not see “objects” — it sees **patterns in numbers**

---

## 🧪 1.7 Small Hands-on Example (Python)

```python
import numpy as np

# Create a simple 3x3 image
image = np.array([
    [0, 100, 255],
    [50, 150, 200],
    [30, 80, 220]
])

print("Image:\n", image)
print("Shape:", image.shape)
```

👉 Output:

```
Shape: (3, 3)
```

---

## 🎯 1.8 Key Takeaways

* Image = grid of numbers
* Pixel = smallest unit
* Grayscale → 1 channel
* Color → 3 channels (RGB)
* Shape tells us size and structure

---

## 🧠 Simple Analogy (to remember forever)

👉 Image is like a **chessboard of numbers**
👉 CNN learns patterns by looking at small squares of that board
